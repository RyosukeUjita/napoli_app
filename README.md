# 要件定義・仕様書：ピザ電話接客シミュレーター (Ver 1.1)

デリバリーピザの電話注文受付における、注文前の基本フロー（挨拶から注文受付の手前まで）を習得するための、フロントエンド単体動作型Webアプリの仕様書です。

---

## 1. 全体システム概要

- **システム名:** デリバリーピザ電話接客研修システム (Pizza Order Quest)
- **目的:** アルバイトスタッフの初期電話応対、正確な顧客情報（番号・名前・住所）の聞き取りおよびマニュアルに沿った確実な復唱スキルの向上。
- **ターゲット:** 新人アルバイトスタッフ
- **動作環境:** PCおよびタブレットのブラウザ（フロントエンド単体、外部データベース不要のMock動作）
- **デザイン方針:**
  - 企業の公式eラーニングを意識した「シック」「クリーン」「信頼感」のあるUI。
  - メインカラーは濃いネイビー（`#0f172a / slate-900` や `#172554 / blue-950` 等）を基調とする。
  - 派手なゲーム的演出（ドット絵など）は避け、実務シミュレーターとしての外観を維持する。

---

## 2. 画面構成および状態管理 (State)

画面遷移は行わず、単一コンポーネント内での状態（State）の切り替えによって3つのフェーズを表現する。

### 2.1. 開始前（初期画面）
- タイトル「デリバリーピザ電話接客研修」と、概要「〜適切な言葉遣いと正確な受注管理の習得〜」を表示。
- 中央に「研修を開始する」ボタン（フラットデザイン、ホバー効果あり）を配置。

### 2.2. 研修中（メイン画面）
画面を上下に3分割してレイアウトする。
1. **上部（ステータスエリア）:**
   - 現在の進行度（例：「Step 2 / 8：用件の誘導」）
   - お客様満足度メーター（初期値100%。クイズ形式のステップで間違えると各ステップの `penalty` 分減点、下限は0%）
2. **中部（対話ログエリア）:**
   - 電話の向こうのお客様のセリフ、またはマニュアルからの指示文をシンプルなダイアログボックスで表示。
3. **下部（操作・解答エリア）:**
   - **タイプが `quiz` の場合:** 4つの選択肢ボタンを表示。間違えた場合はその場で「不適切な応対です。もう一度選択してください」と赤字のエラーメッセージを表示し、該当の `penalty` 分を減点。正解を選ぶまで次へ進ませない。
   - **タイプが `read` の場合:** 4つのボタンの代わりに、「マニュアル通りに口頭で復唱・対応できたら『次へ』を押してください」という趣旨のメッセージと、**「復唱しました（次へ）」ボタン**を1つだけ表示する。

### 2.3. 研修修了（リザルト画面）
- 全8ステップをすべてクリア、または満足度が0%になった時点で自動で切り替わる。
- 「研修修了」の文字、および最終的な「お客様満足度（スコア）」を表示。
- 「もう一度受講する」ボタンを配置し、クリックで初期状態（満足度100%、Step 1、顧客データを再度ランダム抽選）に戻る。
